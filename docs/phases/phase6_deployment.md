# Phase 6: 阿里云部署

**状态**: ⬜ 规划中  
**前置条件**: Phase 5 完成

## 目标

将 Tally 后端部署到阿里云服务器，使 Android App 可以通过公网访问后端服务。

## 待讨论事项

### 1. 服务器配置
- [ ] 选择 ECS 实例规格（CPU/内存/带宽）
- [ ] 选择操作系统（推荐 Ubuntu 22.04 LTS）
- [ ] 配置安全组规则（端口 3000 或反向代理 80/443）

### 2. 部署方式
- [ ] 直接部署 Node.js + PM2
- [ ] 或使用 Docker + Docker Compose
- [ ] 是否需要 Nginx 反向代理

### 3. 数据库
- [ ] 使用阿里云 RDS PostgreSQL
- [ ] 或自建 PostgreSQL（Docker）
- [ ] 数据库备份策略

### 4. 域名与 HTTPS
- [ ] 是否购买域名
- [ ] 配置 SSL 证书（Let's Encrypt 或阿里云免费证书）

### 5. 环境变量管理
- [ ] 服务器上的 `.env` 文件管理
- [ ] KIMI_API_KEY、TAVILY_API_KEY 等敏感信息安全存储

### 6. CI/CD（可选）
- [ ] 是否配置 GitHub Actions 自动部署
- [ ] 或使用阿里云效等其他方案

## 预算评估

| 项目 | 预估费用 |

| RDS PostgreSQL | ~30元/月（最低配） |
